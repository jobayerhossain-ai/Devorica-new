# DEVORICA LAYOUT SYSTEM

Version: 1.0

Status: Core Documentation

This document defines how every page, section, and component should be structured.

Layout is not decoration.

Layout is communication.

Good layout reduces cognitive load.

Great layout builds trust before users read a single word.

---

# DESIGN PRINCIPLE

Every layout should answer three questions:

1. What should users notice first?
2. Where should users look next?
3. What action should users take?

Never place elements randomly.

Every element must have a visual purpose.

---

# AI SELF REVIEW

Before generating any page ask:

✓ Is the hierarchy obvious?

✓ Does the layout guide the eye?

✓ Is whitespace generous?

✓ Does every section have purpose?

✓ Is there visual rhythm?

✓ Does it feel handcrafted?

✓ Does it feel premium?

✓ Would a business owner trust this company after 5 seconds?

✓ Does this look like an international agency rather than a local template?

If any answer is NO,

redesign the layout before generating code.
